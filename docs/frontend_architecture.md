# Архитектура Frontend (PWA)

## 1. Обзор
Одностраничное приложение (SPA) с возможностями Progressive Web App (PWA).
- **Фреймворк:** React 18+
- **Сборщик:** Vite
- **Язык:** TypeScript
- **UI Kit:** Material UI (MUI) v5+
- **State Management:** Zustand (клиент), TanStack Query (сервер)
- **Карты:** @yandex/ymaps3-sdk-private (или react-yandex-maps v2 для стабильности)

## 2. Структура проекта (Feature-Based)
Используем подход, группирующий файлы по функциональным доменам (features), а не по типу файлов.

```text
src/
├── app/                  # Глобальная конфигурация приложения
│   ├── providers/        # Контексты (Auth, Theme, QueryClient)
│   ├── styles/           # Глобальные стили, тема MUI
│   ├── router/           # Настройка роутера (React Router DOM)
│   └── main.tsx          # Точка входа
├── pages/                # Страницы (маршруты)
│   ├── auth/             # Login, Register, ResetPassword
│   ├── home/             # Афиша (публичная)
│   ├── events/           # Детальная страница события
│   ├── profile/          # Личный кабинет (студент/организатор)
│   ├── organizer/        # Кабинет организатора (создание событий)
│   ├── admin/            # Админ-панель
│   └── not-found.tsx
├── features/             # Бизнес-логика и виджеты
│   ├── auth-by-login/    # Формы входа/регистрации
│   ├── event-list/       # Список событий с фильтрами
│   ├── event-map/        # Компонент карты Яндекс
│   ├── qr-scanner/       # Сканер QR для чек-ина
│   ├── certificate-upload/ # Загрузка сертификатов (орг)
│   └── notifications/    # Логика Push (опционально)
├── entities/             # Бизнес-сущности (типы, API хуки, UI карточки)
│   ├── user/             # Типы пользователя, hooks useUser
│   ├── event/            # Типы события, hooks useEvents
│   ├── organization/     # Типы организации
│   └── certificate/      # Типы сертификатов
├── shared/               # Переиспользуемый код без бизнес-логики
│   ├── api/              # Axios инстанс, интерцепторы, типы ответов
│   ├── components/       # UI кит (Button, Input, Modal - обертки над MUI)
│   ├── hooks/            # Общие хуки (useDebounce, useLocalStorage)
│   ├── lib/              # Утилиты (форматирование дат, валидация)
│   └── assets/           # Иконки, шрифты, статика
├── widgets/              # Крупные блоки страниц
│   ├── header/           # Шапка с навигацией
│   ├── footer/           # Подвал
│   ├── event-card/       # Карточка события (полная)
│   └── profile-sidebar/  # Боковое меню профиля
└── vite-env.d.ts
```

## 3. PWA Конфигурация
Реализуется через `vite-plugin-pwa`.

### 3.1. Manifest (`manifest.webmanifest`)
- `display`: standalone
- `theme_color`: #1976d2 (Primary цвет вуза)
- `icons`: набор для iOS и Android (маскируемые и обычные)
- `short_name`: UniEvents
- `categories`: education, lifestyle

### 3.2. Service Worker (Workbox)
Стратегии кэширования:
1.  **Static Assets (JS, CSS, Images):** `CacheFirst` - кэшируем навсегда, обновляем при новой версии.
2.  **API Requests (GET /events):** `NetworkFirst` с таймаутом 3 сек -> падает в кэш (TTL 5 мин).
3.  **API Requests (POST/PUT):** `NetworkOnly` + очередь повторных попыток (Background Sync) для офлайн-регистрации.
4.  **Offline Page:** Отдельная страница `offline.html` с кнопкой "Обновить".

### 3.3. IndexedDB
Используем библиотеку `idb` для хранения:
- Текущего профиля пользователя (для быстрого доступа).
- Черновиков созданных мероприятий (если связь пропала во время ввода).
- Очереди действий (Action Queue) для отправки при появлении сети.

## 4. Маршрутизация и Роли
Используем `react-router-dom` v6+.

### Защищенные роуты (Protected Routes)
Компонент-обертка, проверяющий роль в токене/Store.
- `/admin/*` → Доступ только `ADMIN`.
- `/organizer/*` → Доступ `ORGANIZER`, `ADMIN`.
- `/profile` → Доступ `STUDENT`, `ORGANIZER`, `ADMIN`.
- `/` (Афиша) → Публичный доступ.

Логика редиректа:
- Если гость пытается зайти в ЛК → Редирект на `/login` с `returnUrl`.
- Если студент пытается зайти в админку → Редирект на `/403` или главную.

## 5. Управление состоянием

### Серверное состояние (TanStack Query)
Все данные с бэкенда (события, профиль, список участников) управляются здесь.
- **Кэширование:** `staleTime: 5 * 60 * 1000` (5 минут).
- **Персистентность:** Плагин `persistQueryClient` сохраняет кэш в `localStorage`/`IndexedDB` для мгновенной загрузки при следующем визите.
- **Мутации:** `onMutate` для оптимистичного обновления UI (например, лайк или регистрация), `onError` для отката.

### Клиентское состояние (Zustand)
Глобальное состояние, не требующее перезапроса к серверу.
- `authStore`: Токен, роль, статус авторизации.
- `uiStore`: Открыто ли мобильное меню, тема (светлая/темная), тосты уведомлений.
- `offlineStore`: Статус соединения (online/offline).

## 6. Интеграция с Яндекс.Картами
Компонент `EventMap`:
- Lazy loading скрипта карт (чтобы не тормозить First Paint).
- Отображение метки мероприятия по координатам `lat`, `lng`.
- Клик по метке → открытие балуна с названием и адресом.
- Для мобильных: кнопка "Построить маршрут" (открывает приложение Яндекс.Навигатор или карты через deep link `yandexmaps://`).

## 7. Безопасность на клиенте
- **XSS:** Санитизация всего пользовательского контента перед рендером (MUI делает это автоматически, но осторожно с `dangerouslySetInnerHTML`).
- **Токены:** Хранение `access_token` в памяти (переменная), `refresh_token` в `httpOnly` cookie (если бэкенд поддерживает) или в зашифрованном виде в памяти с осторожностью. *Рекомендация: хранить в памяти, при перезагрузке запрашивать новый через refresh эндпоинт.*
- **CSRF:** Автоматическая подстановка токенов через интерцептор Axios.
- **CSP:** Настройка заголовков на стороне сервера, но проверка в мета-тегах.

## 8. Производительность
- **Code Splitting:** `React.lazy` + `Suspense` для всех страниц (`pages/*`) и тяжелых компонентов (Карты, Редактор описания).
- **Image Optimization:** Использование формата WebP, lazy loading изображений мероприятий.
- **Bundle Analysis:** Визуализация размера бандла (`rollup-plugin-visualizer`) для контроля зависимостей.
- **Target:** Mobile-first CSS, критические стили инлайнятся.

## 9. Локализация (i18n)
- Библиотека `react-i18next`.
- Структура ключей: `feature.component.element` (например, `event.card.register_btn`).
- Форматирование дат: `Intl.DateTimeFormat('ru-RU')`.
- Валюты и числа: `Intl.NumberFormat`.

## 10. Тестирование
- **Unit:** Vitest + React Testing Library (тестирование хуков и утилит).
- **Component:** Скриншотные тесты (опционально).
- **E2E:** Playwright (сценарии: регистрация, создание события, чек-ин).
